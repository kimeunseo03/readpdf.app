# Parser v2 변경 내역

## 목적

출력 화면 검토 결과, 1차 파서는 PDF 텍스트 추출은 성공했지만 일부 필드에서 권리자/소유자 주소나 전체 건물 면적을 잘못 참조했다. v2는 등본 상단 표제부 주소와 전유부분 섹션을 우선 소스로 삼도록 파싱 범위를 제한한다.

## 수정 내용

### 1. 주소 추출

- `등기사항전부증명서...` 헤더 전체를 주소로 잡지 않도록 변경
- `[집합건물] ... 제OO동 제OO층 제OO호` 구간만 `addressRaw`로 추출

### 2. 건물명 추출

- 우선순위 1: 상단 `[집합건물]` 주소 라인에서 건물명 추출
- 우선순위 2: 건물명이 상단 주소에 없는 경우, 동일 동/호가 포함된 도로명주소 괄호에서만 보조 추출
- 권리자/소유자 주소에 포함된 다른 아파트명은 배제

### 3. 동/층/호수 추출

- 상단 주소의 `제OO동 제OO층 제OO호` 패턴만 사용
- `floor` 필드도 함께 채움

### 4. 전유면적 추출

- `【 표 제 부 】 ( 전유부분의 건물의 표시 )` 이후 섹션에서만 추출
- `제OO층 제OO호` 주변의 면적을 우선 사용
- 500㎡ 초과 값은 전체 건물/층 면적 오탐 가능성이 높아 자동 확정하지 않음

### 5. 대지권 비율 추출

- `대지권의 표시` 섹션으로 범위 제한
- `소유권대지권 N분의 M` 패턴 우선
- PDF 텍스트 레이아웃 때문에 날짜가 중간에 끼는 형태도 보정

## 확인 필요

- 실제 등본 PDF 원본으로 재업로드 테스트 필요
- OCR PDF는 아직 실제 OCR 미구현 상태이며, 필요 여부만 감지
- 복합건물/상가 혼합 등본은 전유면적 수동검토가 필요할 수 있음
